# euolaVM 指令集设计
本文将分章描述每个指令的设计。有些指令可能在同一章介绍。

## SetConstant 与 DynSetConstant
这两个指令都用于将 SIL 上某个槽的值设为一个常量，但它们存在一定的区别：
 - SetConstant 在解析期构造。这意味着它更快，但当应用在引用类型上时，即使执行多次，依然只会创建对同一个对象的不同引用，且这个对象永远不会被释放。出错不会持续到运行时。
 - DynSetConstant 在这条指令被执行时构造。它比SetConstant慢一些，但当应用在引用类型上时，每次执行都会创建一个新的对象。如果出错，产生`raw::fatal::dynset_error`中断。

## IsNull
这个指令用于判断一个值是否为 NULL 。该指令仅适用于引用类型；对值类型使用此指令会造成未定义的行为。

## GetStatic
获取一个位于公共变量区的变量。如果试图获取一个不存在的变量，将会产生中断`raw::fatal::static_not_found`。

## SetStatic
设置一个位于公共变量区的变量。如果这个变量不存在，创建；如果存在，则修改。

## GetField
获取一个对象的某个字段。该对象应该是“用户定义对象”，而不是`raw`中的宽松对象。如果字段不存在，产生中断`raw::fatal::out_of_range`。

## SetField
设置一个对象的某个字段。该对象应该是“用户定义对象”，而不是`raw`中的宽松对象。如果字段不存在，创建；如果存在，则修改。
